# Turning AI work into cards

FSRS and spaced repetition are the proven retention mechanism. AI-assisted
work creates more to learn, including for vibe coders and people managing large
projects. Reanthesis' North Star is to bring your AI in as a partner that
organizes cards, so you can focus on studying rather than sorting notes.

## After a meaningful chunk

When a task, investigation, design decision, or other meaningful chunk closes,
pause before the context evaporates. Pull out lessons that should still help
after the current project changes:

- **Facts** — a stable behavior, relationship, constraint, or definition
- **Decisions** — the chosen trade-off and the reason it won
- **Mistakes** — a wrong assumption, failed approach, or misleading signal
- **Patterns** — a reusable way to recognize or handle a class of problem
- **Next-time rules** — a check or action that prevents repeat pain

Treat the transcript as raw material, not as a card source. Confirm the lesson,
remove narrative, and write one atomic retrieval target per card. The answer
should be short enough to check quickly but complete without the original
conversation.

## Durable knowledge filter

Separate knowledge from project context. Durable knowledge remains useful when
the repository, customer, team, or implementation changes. Project context is a
temporary path, ticket, owner, deadline, current status, local workaround, or
other detail whose value expires with the work. Keep that context in the
project's docs or issue tracker unless the user explicitly wants a temporary
runbook deck. When in doubt, prefer the lesson that generalizes beyond the
original project.

## Privacy filter

Never put secrets, credentials, client or customer data, private context, or
conversation-only details into a card. Strip identifying names and proprietary
payloads; replace them with the smallest general description that preserves the
lesson. Do not use a card to preserve a transcript or an unverified claim.

## Quality check

Before proposing a batch, ask:

1. Is there one clear thing to retrieve?
2. Will this still be useful after the project changes?
3. Is the wording self-contained, concise, and specific?
4. Does it avoid private or ephemeral context?
5. Does an existing card already cover it?

Discard candidates that fail. Reuse the collection's deck and tag vocabulary,
then propose the small batch for approval unless the user asked for immediate
import. Create through the normal card-craft workflow and leave the user with
the study queue, not a pile of unfiltered notes.
